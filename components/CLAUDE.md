# CLAUDE.md — components/

## Learnings (agent-maintained)
- Restoring persisted client state (sessionStorage/localStorage) in a client component must happen in a mount `useEffect`, NOT a lazy `useState` initializer — client components still SSR their initial HTML with the default state, so reading storage pre-hydration causes a React hydration mismatch. Related shadowing trap: `import { type KeyboardEvent } from "react"` rebinds the name, so a `window.addEventListener("keydown", …)` handler must type its event `globalThis.KeyboardEvent` (same for TouchEvent/MouseEvent) or it silently typechecks against the wrong (React synthetic) type.
- `crypto.getRandomValues` is NOT secure-context-gated (unlike `crypto.randomUUID`, which is undefined over the plain-http LAN transport this app serves on) — so it's the right client-side random fallback when a verify gate greps the file for `Math.random` (test-runner.tsx `newClientEventId`, wave12b-12). `lib/client/track.ts` still uses the Math.random fallback; fine until a gate targets that file.
- Client-side pre-validation of a server-action form: `event.preventDefault()` in the form's `onSubmit` DOES stop React 19 from invoking `action={serverAction}` — so `<form action={a} onSubmit={h} noValidate>` is the pattern for Ukrainian (or any custom) constraint messages: `h` reads each control's ValidityState and blocks+renders messages (`lib/client/form-errors.ts` `useUkValidation()` hook + `Field`'s `error`/`minLength` props render `role="alert"` + aria-describedby; verified in-browser wave12b-15). Spec gotcha: `validity.tooShort` only fires on USER-edited (dirty) values — a programmatic `.value=` set never trips it, so browser-driving a tooShort assert must `type` into the field, not eval-set it.
- Copy-gated Ukrainian strings containing an apostrophe (з'явиться, обов'язки) as JSX TEXT trip eslint `react/no-unescaped-entities` — but escaping to `&apos;` breaks the verify gate's literal `grep -qF` on the source. Wrap the copy in a JS string expression instead (`<span>{"…з'явиться…"}</span>`): lint-clean AND the literal bytes stay greppable (bit wave13-12).
- The shell's idle-freeze (`glass-shell.tsx` adds `.bg-idle` to `<html>` after ~1.2s of no input; `app/globals.css`: `.bg-idle * { animation: none !important }`) PAUSES every CSS `@keyframes` animation while the user sits still. So any INTENTIONAL infinite/ambient loop that must keep running during no-interaction (e.g. the pre-exam calm-ritual breathing pacer, `components/calm-ritual.tsx`) must use the **Web Animations API** — `el.animate([...], {iterations: Infinity})` — NOT a CSS animation: WAAPI animations are independent of the CSS `animation` property, so `.bg-idle`'s `!important` override can't stop them. (Also: gate the WAAPI loop off `prefers-reduced-motion` in JS and `anim.cancel()` on cleanup.) React 19: passing `autoFocus` to the `ui.tsx` `Button` (it spreads `...props` onto the DOM `<button>`) focuses it on mount — simpler than a forwarded ref for focusing a freshly-mounted overlay control.
- The `test-runner.tsx` answer options are ARIA `role="radio"` BUTTONS inside a `role="radiogroup"` (test-runner.tsx:518/537), NOT native `<input type="radio">`. So a DOM count/query in a browser-audit eval must use `document.querySelectorAll("[role=radio]")` — `input[type=radio]` returns 0 (false FAIL) even though agent-browser `find role radio click` answers fine (it targets the ARIA role). Also: «Завершити тест» (setConfirming) only renders when `isLast || isExam || isMarathon` — for a NON-exhausting practice mode (QUICK/SIGN_TRAINER/TOPIC) it appears ONLY on the last question, so a browser-audit finish must advance via «Далі» to the last question first when the finish button is absent (bin/browser-audit.sh `finish_session`, wave15-15).
- Phone chrome composition: `AppNav` returns `null` on `/test/` routes (app-nav.tsx — "the runner owns the screen"), so runner-level `sticky bottom-0` chrome can't collide with the tab capsule; EVERYWHERE ELSE the capsule is `fixed inset-x-0 bottom-0 z-40` on phone and the (app) layout pads `pb-28 sm:pb-6` to clear it — any new bottom-anchored element on a non-test page must sit above ~112px or hide the capsule. Tailwind padding-reset gotcha: a base `pb-[…]` arbitrary utility is emitted AFTER the `p`/`px`/`py` groups, so `sm:p-0`/`sm:py-0` do NOT override it — add an explicit `sm:pb-0`. Glass classes set `position:relative` in `@layer components`, but the `sticky` utility (utilities layer) wins, so `glass-e1 sticky top-0` composes fine.
