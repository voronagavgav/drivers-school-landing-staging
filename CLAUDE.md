- Adding `"use client"` to a component in the `(app)` shell (e.g. `components/app-nav.tsx` for `usePathname`) pulls its entire import graph into the CLIENT bundle. Importing `@/lib/rbac` (or `@/lib/auth`) client-side → Turbopack build error `the chunking context does not support external modules (request: node:module)` because rbac→auth transitively pulls bcrypt/`node:module`. Fix: DON'T call server-graph helpers (`canManageContent`, `getCurrentUser`) in a client component — compute the result server-side (in the layout/page) and pass a PLAIN prop (`canManage: boolean`, `userName: string`). Wiring a `lib/server/*` action (which is `server-only`, NOT a `"use server"` file, so a client component can't import it) into a client form: define module-scope inline-`"use server"` wrapper fns in the SERVER page (`async function submit(_prev: State|null, formData: FormData) { "use server"; return realAction(formData); }` — adapting to the `useActionState` `(prev, payload)` signature) and pass them to the client form component as a prop; the client file may `import type` the state shape only. - ELO/RASCH FOLD ORDER-SENSITIVITY (wave22-01, `scripts/oracles/gen-wave22-oracles.py`): the online Elo update (θ+=K·e, β−=K·e, guess-adjusted `P=g+(1−g)·σ(θ−β)`, `g=min(1/optionCount,0.45)`, adaptive `K(n)=0.4/(1+n/20)` pre-increment) is NOT order-invariant in general, BUT a "shuffle" that RE-BLOCKS by a different outer axis (user-outer → item-outer) provably leaves EVERY final β/θ IDENTICAL — it preserves both the per-item user-order AND the per-user item-order, so each user's θ when it hits a given item is unchanged (that user has answered exactly the same prior items). To actually demonstrate order sensitivity you must change a RELATIVE order — REVERSE the stream (or otherwise permute within a user's/item's sub-sequence); a re-block will false-pass a `differs=true` oracle as `differs=false`. So the wave22-03/04/07 TS order-sensitivity vectors must use the REVERSED-stream permutation (first β divergence at q0, |Δβ|≈0.074 on the fixed 5u×8i stream), never an item-outer re-block. Also: verify.sh anchors like `order_sensitive differs=true` / `k_decay monotone=true` are grepped as CONTIGUOUS substrings — put the boolean immediately after the label in the print, not after intervening `item=`/`deltas=` fields.
